## 🔧 Configuration Required

### For Next.js 15 Users

Add to your `next.config.js`:

```javascript

const nextConfig = {
  transpilePackages: ['divt-text-editor'],
};

module.exports = nextConfig;
```

### For Next.js 13-14 Users

No configuration changes required, but the transpilePackages setting is recommended.

## 📝 Breaking Changes

### 1. Dependencies Now Externalized

**What Changed**: All Tiptap and Lucide React dependencies are now externalized.

**Impact**: Your project must have these dependencies installed (usually automatic via npm).

**Action Required**: None if installing via npm. The package manager will handle dependencies.

**Why**: Reduces bundle size by 97% and prevents duplicate React instances.

### 2. Next.js 15 Requires Configuration

**What Changed**: Next.js 15 requires the `transpilePackages` configuration.

**Impact**: Editor won't work without this configuration in Next.js 15.

**Action Required**: Add `transpilePackages: ['divt-text-editor']` to `next.config.js`.

**Why**: Rolldown bundler needs explicit transpilation for external packages.

### 3. React 19 Support Added

**What Changed**: Peer dependencies now accept React 18 or 19.

**Impact**: You can use the editor with React 19.

**Action Required**: None.

**Why**: Future-proofing for React 19 adoption.

## 🐛 Bug Fixes

### Fixed: Module Resolution with Rolldown

**Issue**: Next.js 15's Rolldown bundler couldn't resolve the package.

**Solution**: Externalized all dependencies and added proper module exports.

### Fixed: Duplicate React Instances

**Issue**: React was bundled in the package, causing duplicate instances.

**Solution**: Made React a peer dependency only.

### Fixed: Large Bundle Size

**Issue**: Package was 580KB due to bundled dependencies.

**Solution**: Externalized dependencies, reducing size to 18KB.
